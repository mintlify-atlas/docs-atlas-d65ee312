---
title: "Credential Management Commands"
description: "VICI commands for loading and managing certificates, private keys, and shared secrets."
---

These commands manage the credential store of the running daemon. Credentials loaded over VICI are tracked separately from those read from the filesystem at startup. Use `clear-creds` to remove all VICI-loaded credentials at once.

## load-cert

Load a certificate into the daemon.

**Input**

<ParamField body="type" type="string" required>
  Certificate type. One of: `X509`, `X509_AC`, `X509_CRL`.
</ParamField>

<ParamField body="flag" type="string">
  X.509 certificate flag. One of: `NONE`, `CA`, `AA`, `OCSP`.
</ParamField>

<ParamField body="data" type="string" required>
  PEM or DER encoded certificate data.
</ParamField>

**Response**

<ResponseField name="success" type="string">
  `yes` on success, `no` on failure.
</ResponseField>

<ResponseField name="errmsg" type="string">
  Human-readable error description on failure.
</ResponseField>

---

## load-key

Load a private key into the daemon.

**Input**

<ParamField body="type" type="string" required>
  Private key type. One of: `rsa`, `ecdsa`, `ed25519`, `ed448`, `any`.
</ParamField>

<ParamField body="data" type="string" required>
  PEM or DER encoded private key data.
</ParamField>

**Response**

<ResponseField name="success" type="string">
  `yes` on success, `no` on failure.
</ResponseField>

<ResponseField name="id" type="string">
  Hex-encoded SHA-1 identifier of the corresponding public key. Present on success and used for subsequent `unload-key` calls.
</ResponseField>

<ResponseField name="errmsg" type="string">
  Human-readable error description on failure.
</ResponseField>

---

## unload-key

Unload the private key with the given identifier. Only keys loaded over VICI can be unloaded.

**Input**

<ParamField body="id" type="string" required>
  Hex-encoded SHA-1 key identifier of the private key to unload. Obtain this from the `id` field returned by `load-key` or `load-token`.
</ParamField>

**Response**

<ResponseField name="success" type="string">
  `yes` on success, `no` on failure.
</ResponseField>

<ResponseField name="errmsg" type="string">
  Human-readable error description on failure.
</ResponseField>

---

## get-keys

Return identifiers of private keys loaded exclusively over VICI. Keys loaded by other backends are not included.

**Input**

No input parameters.

**Response**

<ResponseField name="keys" type="list">
  List of hex-encoded SHA-1 key identifiers.
</ResponseField>

---

## load-token

Load a private key stored on a PKCS#11 hardware token into the daemon. Keys loaded this way can be listed and unloaded using `get-keys` and `unload-key`.

**Input**

<ParamField body="handle" type="string" required>
  Hex-encoded `CKA_ID` attribute of the private key object on the token.
</ParamField>

<ParamField body="slot" type="string">
  Optional PKCS#11 slot number.
</ParamField>

<ParamField body="module" type="string">
  Optional PKCS#11 module (shared library path or configured module name).
</ParamField>

<ParamField body="pin" type="string">
  Optional PIN to access the token. If not provided, the PIN must be supplied by another means (e.g., a PIN callback configured in the PKCS#11 plugin).
</ParamField>

**Response**

<ResponseField name="success" type="string">
  `yes` on success, `no` on failure.
</ResponseField>

<ResponseField name="id" type="string">
  Hex-encoded SHA-1 identifier of the public key, derived from the token key.
</ResponseField>

<ResponseField name="errmsg" type="string">
  Human-readable error description on failure.
</ResponseField>

---

## load-shared

Load a shared secret (PSK, EAP password, XAuth password, or NTLM hash) into the daemon.

**Input**

<ParamField body="id" type="string">
  Optional unique identifier for this shared key. Used to unload it later with `unload-shared`.
</ParamField>

<ParamField body="type" type="string" required>
  Shared key type. One of: `IKE` (PSK), `EAP`, `XAUTH`, `NTLM`.
</ParamField>

<ParamField body="data" type="string" required>
  Raw shared key data.
</ParamField>

<ParamField body="owners" type="list">
  List of IKE identity strings that own this shared key. Used to match the key to specific peers. Omit to allow any peer to use the key.
</ParamField>

**Response**

<ResponseField name="success" type="string">
  `yes` on success, `no` on failure.
</ResponseField>

<ResponseField name="errmsg" type="string">
  Human-readable error description on failure.
</ResponseField>

**Python example**

```python
v.load_shared({
    "type": "IKE",
    "data": "s3cr3t",
    "owners": ["moon.strongswan.org", "sun.strongswan.org"]
})
```

## load-authority

Load a single certification authority (CA) definition into the daemon. An existing authority with the same name is replaced.

The input message contains a top-level section named after the CA. The contents follow the `swanctl.conf(5)` authority format.

**Input**

<ParamField body="<authority name>" type="section" required>
  A section named after the CA. Refer to `swanctl.conf(5)` for the supported parameters (`cacert`, `crl_uris`, `ocsp_uris`, `cert_uri_base`, etc.).
</ParamField>

**Response**

<ResponseField name="success" type="string">
  `yes` on success, `no` on failure.
</ResponseField>

<ResponseField name="errmsg" type="string">
  Human-readable error description on failure.
</ResponseField>

---

## unload-authority

Unload a previously loaded CA definition by name.

**Input**

<ParamField body="name" type="string" required>
  Name of the certification authority to unload.
</ParamField>

**Response**

<ResponseField name="success" type="string">
  `yes` on success, `no` on failure.
</ResponseField>

<ResponseField name="errmsg" type="string">
  Human-readable error description on failure.
</ResponseField>

## flush-certs

Flush the certificate cache. Optionally restrict flushing to a specific certificate type.

**Input**

<ParamField body="type" type="string">
  Certificate type to flush. One of: `X509`, `X509_AC`, `X509_CRL`, `OCSP_RESPONSE`, `PUBKEY`, or `ANY` (default — flushes all types).
</ParamField>

**Response**

<ResponseField name="success" type="string">
  `yes` on success, `no` on failure.
</ResponseField>

<ResponseField name="errmsg" type="string">
  Human-readable error description on failure.
</ResponseField>

---

## clear-creds

Clear all certificates, private keys, and shared keys loaded over VICI. This also flushes the credential cache.

<Warning>
This command removes all VICI-loaded credentials. Credentials loaded by other backends (e.g., from the filesystem) are not affected, but the shared credential cache is flushed for all backends.
</Warning>

**Input**

No input parameters.

**Response**

<ResponseField name="success" type="string">
  `yes` on success, `no` on failure.
</ResponseField>

<ResponseField name="errmsg" type="string">
  Human-readable error description on failure.
</ResponseField>
